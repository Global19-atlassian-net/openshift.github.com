OpenShift Origin Controller - Authentication Plugin

The authentication plugin module allows you to plugin in custom authentication backends into OpenShift Origin.

OpenShift Origin currently provides 2 modes of authentication:

  • Mongo Auth plugin: A simple authentication plugin which stores usernames and hashed password in Mongo DB.
  • Remote-user Auth plugin: A apache based authentication which allows you to use any apache authentication module with OpenShift. This supports basic user auth and Kerberos based authentication amongst other options.

Building a new plugin

An OpenShift Authtication plugin is a Rails Engine which includes a library class which extends OpenShift::AuthService. The plugin class must implement either:

  • authenticate(login, password). Refer to the Mongo auth plugin for an example.
  • authenticate_request(controller). Refer to the Remote-user plugin for an example.
