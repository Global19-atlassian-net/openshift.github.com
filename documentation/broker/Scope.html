Overview

A scope represents a constraint on an authenticated user's access to the system. The actions a given authenticated user can perform are the union of actions that the active scopes allow - for instance, a client with the “read” scope and the “domain/<id>/admin” scope has read access to all resources visible to the active user, and full administrative access on the identified domain. If the active user is himself limited to a set of actions on that domain, the scope does not grant any more actions than he has normally.

def self.for(s)
  return nil if s.blank?
  scopes[s] || scopes[nil].find do |scope|
      r = scope.matches?(s)
      return scope.new(r) if r
    end || nil
end

def self.list(spec)
  Scopes.new.concat(
    (spec || '').split(/[,\s]/).
      map!(&:strip).
      map!(&:downcase).
      uniq.
      map!{ |s| self.for(s) }.
      compact
  )
end

def self.load_scopes(classes)
  classes.inject({nil => []}) do |h, c|
    scope_class = c.constantize
    if scope_class.respond_to?(:matches?)
      h[nil] << scope_class
    else
      h[scope_class.scope_name] = scope_class.new
    end
    h
  end
end

def self.Scopes(arg)
  case arg
  when Scopes then arg
  when String then Scope.list!(arg)
  when Array then Scopes.new.concat(arg)
  else Scopes.new
  end
end
